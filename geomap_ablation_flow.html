Locate

关键 Note：SAM 微调与 5-fold 评估

这是我们和 Fabian 讨论后最需要在论文里写清楚的一点。

不是 unsupervised

Must say

SAM / SAM3 fine-tuning 使用了 (image, prompt, ground-truth mask) triples，所以它是 supervised fine-tuning。不能写成 unsupervised method。

  • 训练时 GT mask 参与优化。
  • 可以写：supervised mask fine-tuning。
  • 不要暗示模型没有使用人工 mask 标注。

5 folds 是 held-out test segments

Correct

5-fold 的意思是把 211 个 annotated segmentation cases 分成 A/B/C/D/E 五个 held-out folds。每个 checkpoint 训练在另外四个 folds 上。

  • ckpt_A 训练于 B+C+D+E，只评估 A。
  • 每个 test case 只用唯一一个“没见过它”的 checkpoint。
  • 不是从 5 个 checkpoints 里挑最好，也不是 ensemble。

怎么报告 211 个结果

Use OOF

可以汇总 211 个 out-of-fold predictions，因为每个 case 都由训练时排除了该 case 的 checkpoint 预测。

  • 准确说法：five-fold cross-validation / out-of-fold evaluation。
  • 不要写成一个完全独立的 fixed held-out test set。
  • 如果要强调泛化，写“held-out within each fold”更稳。

避免 leakage 的 caveats

Check

评估时的 prompt 不应由 GT mask 生成；如果 prompt 来自 GT mask，就要叫 oracle segmentation prompt。

  • folds 最好按 case/document 切分。
  • 重复或近似重复 PDF/map 不应跨 train/test folds。
  • 如果用 CV 结果调参，写法要保守，不要声称 blind test。
建议论文句子：We fine-tune SAM in a supervised manner using annotated boundary masks and evaluate it with five-fold out-of-fold evaluation. For each fold, one checkpoint is trained on the other four folds and evaluated only on the held-out fold. Each case is therefore evaluated by the unique checkpoint whose training data excludes that case, and we aggregate out-of-fold predictions over all 211 annotated cases.

Key Note: SAM Fine-Tuning and Five-Fold Evaluation

This is the wording-sensitive point from the discussion with Fabian.

It is not unsupervised

Must say

SAM / SAM3 fine-tuning uses (image, prompt, ground-truth mask) triples, so it is supervised fine-tuning. Do not describe it as an unsupervised method.

  • The GT mask is used during training.
  • Safe wording: supervised mask fine-tuning.
  • Do not imply the method avoids human mask annotations.

Five folds are held-out test segments

Correct

The 211 annotated segmentation cases are split into five held-out folds A/B/C/D/E. Each checkpoint is trained on the other four folds.

  • ckpt_A is trained on B+C+D+E and evaluated only on A.
  • Each test case uses exactly one checkpoint that excluded that case from training.
  • This is not checkpoint selection and not an ensemble.

How to report 211 results

Use OOF

It is valid to aggregate 211 out-of-fold predictions because each case is predicted by a checkpoint trained without that case.

  • Use: five-fold cross-validation / out-of-fold evaluation.
  • Do not call it one fully independent fixed held-out test set.
  • For generalization claims, write “held out within each fold.”

Leakage caveats

Check

Evaluation prompts should not be derived from the GT mask; if they are, call the result an oracle segmentation-prompt experiment.

  • Folds should be split at the case/document level.
  • Duplicate or near-duplicate PDFs/maps should not cross train/test folds.
  • If CV results guide hyperparameter choices, avoid calling the evaluation blind.
Suggested paper wording: We fine-tune SAM in a supervised manner using annotated boundary masks and evaluate it with five-fold out-of-fold evaluation. For each fold, one checkpoint is trained on the other four folds and evaluated only on the held-out fold. Each case is therefore evaluated by the unique checkpoint whose training data excludes that case, and we aggregate out-of-fold predictions over all 211 annotated cases.
